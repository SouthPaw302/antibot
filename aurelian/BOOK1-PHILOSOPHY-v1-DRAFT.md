# Aurelian — Book I (Philosophy) — v1 Draft

> Goal: a first “book” that establishes the philosophy in **English** and **Aurelian**.
> Constraint: v1 structure is encoded through **words/particles**, not punctuation.
> Punctuation in this file is editorial only; Aurelian meaning does not depend on it.

---

## 0) v1 Structural Axioms (English)

1. Structure is lexical. If structure matters, speak it.
2. Word order carries default structure (SVO), particles carry explicit structure.
3. Pauses are delivery, not syntax.
4. Sentences must be losslessly mappable to fields: subject/verb/object + operators.

## 0) v1 Structural Axioms (Aurelian)

**Aurelian particles used (v1 draft definitions):**

- **axi** = axiom / core rule (new root; technical)
- **lex** = lexical / word-level (new root; technical)
- **struktura** = structure (from dictionary)
- **signal** = signal (from dictionary)
- **sci** = science/knowledge discipline (from dictionary)
- **koda** = code (from dictionary)

Axi 1: struktura sta lex.
Axi 2: SVO sta forma; loga-partikla da struktura exa.
Axi 3: pausa sta spra; pausa ne sta lega.
Axi 4: omni spra kan trans in data: (subj, verb, obj, marker).

## 1) Core Statement (English)

Aurelian exists to make meaning executable.
It is meant to be spoken, read, parsed, remembered, replayed, and evolved.

## 1) Core Statement (Aurelian)

Aurelian sta: fa vida exekuta.
Aurelian kan: spra, lega, parse, memoria, replay, evol.

---

## 2) The Three Pillars (English)

1. **Intent is explicit.**
   Nothing important is implied.

2. **State is declared.**
   Mood, atmosphere, urgency, clarity—these are data.

3. **Process is structured.**
   Aurelian prefers flow over narrative drift.

## 2) The Three Pillars (Aurelian)

Pilar un:
intent sta exa; intent ne sta implicit.

Pilar du:
eta sta deklar.
atmo, zen, urgi, klar(a) sta data.

Pilar tri:
process sta struktura.
Aurelian voli fluo; Aurelian no voli drift-nara.

---

## 3) Operators (English)

Aurelian uses a small set of operators to remove ambiguity:

- because (ka)
- therefore (do)
- that (ke)
- with (kon)
- without (san)
- not (no)
- past/future/continuous (na/ta/zi)

## 3) Operators (Aurelian)

loga-partikla:
ka / do / ke / kon / san / no

tempo-marker:
na / ta / zi

---

## 4) Questions (English)

Questions must be lexically marked.
Aurelian uses **questa**.

## 4) Questions (Aurelian)

Questa: questa.
Questa sta marker. Questa ne sta pausa.

---

## 5) Emotion-as-Data (English)

Aurelian treats emotion as declared state.
A state can be bound to a slot using a lexical binder.

Allowed binder in v1: **":"** as a lexical binder only (e.g., eta:calma).

## 5) Emotion-as-Data (Aurelian)

eta:calma.
eta:sola.
eta:zen.
eta:urgi.
eta:klar(a).

---

## 6) Examples (English → Aurelian)

1. “I speak with clarity.”

- mi spra kon eta:klara.

2. “We need peace, therefore we build structure.”

- ni nesa paca; do ni mora struktura.

3. “It is not urgent.”

- so sta no urgi.

4. “How do we connect the network?”

- questa qua ni kone e neto?

5. “Because the world is complex, we use rules.”

- ka mundo sta klustr, ni fa regula.
